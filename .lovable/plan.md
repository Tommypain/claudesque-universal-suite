# خطة إكمال Omega Office Suite

نقل الحزمة من ملف `office_suite_v6.html` (4507 سطر) إلى هيكل React/TanStack مع **الحفاظ الكامل على هوية تصميم Claude** (نفس ألوان، نفس الكبسولات، نفس الأنماط الأربعة، الوضع الداكن)، ثم إكمال الناقص وتلميع التصميم وجعله يعمل على كل منصة.

## المبدأ الأساسي
لن نغيّر روح التصميم. كل قيم الـ CSS (المتغيرات، الكبسولات، الأنماط `basic / liquid-glass / classic / modern`، الوضع الداكن) تُنقل **حرفيًا** كما هي. النقل لـ React هو إعادة تنظيم للكود فقط، مش إعادة تصميم.

## المراحل

### المرحلة 1 — الأساس والتصميم
- نقل كل متغيرات الألوان والأنماط من `<style>` إلى `src/styles.css` كما هي (tokens + 4 layout classes + dark mode).
- تحميل خط Inter وأيقونات Tabler.
- إنشاء سياق حالة عام (`SuiteContext`) يحمل: التطبيق النشط، التاب النشط، النمط (layout style)، الوضع الداكن، الزووم.
- بناء الهيكل العام (App Shell): الشريط الجانبي الأيسر (4 تطبيقات) + الريبون العلوي + شريط الحالة السفلي.

### المرحلة 2 — نظام الريبون
- مكوّن `Ribbon` عام يقرأ التابات حسب التطبيق النشط (File, Home, Insert, Draw, Layout, References, Review, View + تابات Sheet/Impress الخاصة).
- نقل كل مجموعات الأدوات (الكبسولات، أزرار split، القوائم المنسدلة) كمكوّنات React قابلة لإعادة الاستخدام (`RibbonGroup`, `RibbonButton`, `SplitButton`, `RibbonSelect`).
- **إكمال الأدوات الناقصة** بنفس ستايل Claude لكل تطبيق حتى تكون كل تاب ممتلئة ومتسقة.

### المرحلة 3 — التطبيقات الأربعة (Workspaces)
- **Word**: محرر صفحات متعدد + شريط المسطرة + مصغّرات الصفحات الجانبية + إحصائيات + الحفظ التلقائي (localStorage).
- **Sheet**: شبكة الخلايا + شريط الصيغة + تقييم الصيغ (SUM وغيرها) + ألسنة الأوراق السفلية.
- **Impress**: شريط الشرائح الجانبي + منطقة الشريحة + مشغّل العرض ملء الشاشة.
- **PDF**: شريط الصفحات + منطقة التمرير + أدوات التعليق/التوقيع.
- ربط كل أدوات الريبون بمنطق كل تطبيق (تنسيق نص، إدراج، إلخ).

### المرحلة 4 — استيراد/تصدير الملفات
- دمج مكتبات `mammoth` (قراءة docx)، `SheetJS` (xlsx/csv)، `docx` (توليد docx)، `FileSaver` عبر npm بدل CDN.
- استيراد وتصدير المستندات بنفس المنطق الحالي.

### المرحلة 5 — التجاوب (موبايل/تابلت)
- جعل الشريط الجانبي والريبون قابلين للطي على الشاشات الصغيرة.
- ريبون قابل للتمرير أفقيًا + قوائم منسدلة بدل الصفوف على الموبايل.
- ضبط مساحة العمل (المستند/الشبكة/الشريحة) لتناسب 390px وما فوق.

### المرحلة 6 — قابلية التثبيت على كل المنصات
- إضافة `manifest.json` + أيقونات لجعل التطبيق قابلًا للتثبيت (Add to Home Screen) على Windows/Mac/Linux/iOS/Android من المتصفح — يعمل بالكامل في **النسخة المنشورة**.
- توثيق خطوات الحزم native لاحقًا: **Electron** للديسكتوب و**Capacitor** لـ iOS/Android (تُنفَّذ كخطوة منفصلة عبر سكربتات الحزم بعد اكتمال الويب).

## تفاصيل تقنية
- React 19 + TanStack Start، مسار واحد رئيسي `src/routes/index.tsx` يحتوي على الـ Shell، وكل تطبيق كمكوّن داخلي يظهر/يختفي (مطابق لسلوك `workspace-view.active` الحالي).
- المنطق الذي يعتمد على DOM (`contentEditable`, الخلايا) يُنقل لمكوّنات client-side مع `refs` و`useEffect` (لتفادي مشاكل SSR مع `window`/`document`/`localStorage`).
- مكتبات الملفات تُحمَّل client-side فقط.
- لا حاجة لقاعدة بيانات في هذه المرحلة (الحفظ محليًا). يمكن إضافة Lovable Cloud لاحقًا لو أردت حفظ سحابي/حسابات مستخدمين.

## ملاحظات
- معاينة Lovable داخل iframe لا تدعم تثبيت PWA كاملًا؛ التثبيت يعمل في الرابط المنشور.
- الحزم native (Electron/Capacitor) تتطلب خطوة بناء منفصلة بعد إنجاز نسخة الويب.
